 首先，想要通过Hexo和Github Pages搭建个人博客，参考的是[**这个博客**]( http://vor.ink/2018/04/25/Hexo+Github%20Pages%E6%89%93%E9%80%A0%E5%B1%9E%E4%BA%8E%E8%87%AA%E5%B7%B1%E7%9A%84%E5%8D%9A%E5%AE%A2/)。尝试之后，基本完成了该博主的提示，但是对博客目录的划分及写博客等内容不太熟悉，故又寻找其他教程。

第二次参考的是[**这个博客**]( https://cloud.tencent.com/developer/article/1607408)。这个博主允许直接Fork它的博客仓库，这确实很简单，只需要在存放博客的_posts目录上传自己的博客即可，需要注意md文件的命名规范，及文件开头的YAML Front Matter格式的部分(可以输入---然后回车，即可编辑该样式内容)。在写这篇博客的时候，我正在用的就是此种方法。



记录以下参考该仓库的过程：

  首先，在该博主的github仓库Fork，Fork即像叉子一样分流，提交内容也可以提交到主分支上。然后修改仓库名，修改站点配置文件_config.yml的相关内容。

  然后，复制该仓库的ssh地址，在本地用 git clone URL 的命令将仓库下载到本地，此时本地仓库与远程仓库已经关联。

  关于新建博客，可以在_posts文件夹下添加md文件后，利用Git通过git status 、git add . 、git commit -m "message" 、git push origin master的命令同步远程仓库，然后GitHub Pages会自动解析该位置的内容更新博客。
